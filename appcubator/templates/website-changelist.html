Change Log

  • Upgrade from Django 1.4 to 1.5 We updated the generated apps from 1.4 to 1.5 so you can take advantage of the latest and greatest that django has to offer.
  • Change to our user system Previously, we generated apps with 1 User model and 1 Userprofile model, and linked them with a one to one relationship. With 1.5, we’re using a unified user model. Now it’s substantially easier to alter the User model, and import/export user data.
  • New login and signup forms Previously, the generated apps shipped with a username login/signup system by default. We’ve made it possible for people to sign in with their username or email. We simplified the signup form to have only email, name, and password, making it easier for people to sign up for your apps. Note: If you had a login/signup form on your page, you should delete it and drag a new one onto the page from the gallery pane on the right of the editor. Then you will see the new version of the forms.
  • Added “Purple Theme” This new theme features a modern look, with a textured background and sans serif fonts.
  • Improved user edit forms Previously, user edit forms only allowed user to edit additional fields that you added to the user model. Now, users can also edit their name, email, username, and all their other fields. This is useful for creating account settings pages and edit user profile pages.
  • Switched from sqlite to MySQL Previously, generated apps ran with sqlite as the database backend. Due to its limited ability to alter its schema, we switched to MySQL. Now, when you add or remove fields and tables, schema migrations automatically work correctly.
  • Improved downloadable code Previously, our downloaded code was missing a few requirements, and was hard to install. We fixed our requirements file and are working on a better README. Reach out to us if you want us to guide you through the process of running the downloaded code on your computer.
  • Added more Resources
  • Created a Community Page
  • Updated the Website
