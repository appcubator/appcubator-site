Custom Code Integration

Overview

One of the benefits of Appcubator is that it generates developer-friendly source code, so that you can modify it and easily implement custom logic, html, or javascript.

Integration is as easy as pushing to a git repo. When you git push, Appcubator will automatically update your deployment with the code in the master branch.

Basic workflow

You app has a git repo for version control, and a live, deployed copy of your source code. To integrate your custom code, you would:

  1. clone the git repo
  2. change or add to source code in the repo
  3. commit and push your changes to the master branch
  4. make a commit to the master branch and git push

After this point, when you redeploy from the Appcubator web interface (by pressing Test run), Appcubator will safely update your code to reflect your changes in the editor, without overwriting your custom code changes.

If a merge conflict arises when Appcubator tries to generate code, it will save the merge conflict in a separate branch of the repo. You will have the opportunity to mark a resolution, and push your resolution back to master.

Setting up git

In order to be able to clone the git repo, you need to tell us your ssh public key, so can ensure that unauthorized people can't download or modify your code. You can enter your public key(s) in the account settings page here.

We will tell you the repo name

git clone REPO_NAME
