    How to run your Django app (on a Mac)

    In this tutorial, you'll learn how to run a Django app on your computer. This is an important first step if you want to continue to develop your application.

    We'll start with the social network we built in the Building a Social Network Tutorial, and go through the steps required to run the code on your computer.

    After this tutorial, you'll be able to modify the code to produce a finished product, and then deploy your app to the cloud.


    For these tutorials, we'll assume you have a basic knowledge of how to use the Terminal. For a quick tutorial, scroll all the way down.

    Setting up the environment - installing software

    Appcubator apps use the Python/Django/Sqlite stack. You can use the official guide here, or follow our slides, which are beginner-friendly.

    Note: This is a very common process and there's more than one way to do it. If you get stuck at any point, just chat with us or send us an email. We'd be happy to help you out.

    1. Installing Python, pip, and virtualenv

    Below are the summarized notes from this excellent guide.

    First check if you have Python installed, and if so, what version you have.

    $ python --version
Python 2.7.2

    You should have Python 2.7.# installed. If not, you'll need to upgrade it. Contact us if this is the case.

    Now check if you have pip.

    $ pip --version
pip 1.2.1 from /Library/Python/2.7/site-packages/pip-1.2.1-py2.7.egg (python 2.7)

    If you see an output like this instead, you'll need to install pip.

    $ pip --version
-bash: pip: command not found
$ sudo easy_install pip
...

    Now let's make sure pip is up to date

    $ sudo pip install --upgrade pip
Password:
Downloading/unpacking pip from https://pypi.python.org/packages/source/p/pip/pip-1.4.1.tar.gz#md5=6afbb46aeb48abac658d4df742bff714
  Downloading pip-1.4.1.tar.gz (445kB): 445kB downloaded
  Running setup.py egg_info for package pip
...
Successfully installed pip
Cleaning up...

    Now install virtualenv, a tool which makes it easier to cleanly install and remove python packages

    $ sudo pip install virtualenv
Downloading/unpacking virtualenv
  Downloading virtualenv-1.10.1.tar.gz (1.3MB): 1.3MB downloaded
  Running setup.py egg_info for package virtualenv
...
Successfully installed virtualenv
Cleaning up...

    Congratulations! If you got through all this, you've successfully installed Python, pip, and virtualenv, the most popular tools for Python programming.

    Installing SQLite

    Your webapp needs to store data somewhere about the users. We chose the SQLite Database because it's easy to install and gets the job done.

    If you're using a recent version of Mac OS X you should already have this installed. Let's confirm.

    $ sqlite3 --version
3.7.17 2013-05-20 00:56:22 118a3b35693b134d56ebd780123b7fd6f1497668

    If you see something like this, you're good to go! If you see this instead:

    $ sqlite3
-bash: sqlite3: command not found

    then you'll need to install sqlite. Contact us for help on that.

    Congratulations! You now have database software installed! We're almost there.

    Running the code

    Now that you have all the tools installed, we can get the web app source code up and running.

    Download the code

    Download the zip and extract the zip file somewhere memorable, like in your Documents on on your Desktop. Or if you know how to use git, or clone the git repository.

    Install Django and other packages
    $ virtualenv venv
New python executable in venv/bin/python
Installing Setuptools...done.
Installing Pip...done.
    $ source venv/bin/activate
    $ pip install -r requirements.txt
    Create the database
    $ python manage.py syncdb
    $ python manage.py migrate
    Run the web app
    $ python manage.py runserver
